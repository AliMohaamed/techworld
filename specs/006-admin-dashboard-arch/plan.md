# Implementation Plan: Standalone Admin Dashboard Architecture

**Branch**: `006-admin-dashboard-arch` | **Date**: 2026-03-19 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/006-admin-dashboard-arch/spec.md`

## Summary

Build a distinct Next.js Admin Dashboard application physically separated from the Customer Storefront. The workspace will be refactored into a Turborepo monorepo to safely share the Convex backend (`packages/backend`) and UI components (`packages/ui`). The admin dashboard will implement strict RBAC validation via Convex Auth, order lifecycle verification flows (FSM), and catalog management interfaces.

## Technical Context

**Language/Version**: TypeScript / Node 20  
**Primary Dependencies**: Next.js 16.1.6, React 19, Convex ^1.33.1, Tailwind CSS v4, shadcn/ui, Turborepo  
**Storage**: Convex (Backend & Real-Time Database)  
**Testing**: Manual / Future integration  
**Target Platform**: Web (Desktop-first)  
**Project Type**: Monorepo Web Application  
**Performance Goals**: High-concurrency operations, <45s order verification flow  
**Constraints**: Strict logical decoupling (Frontend is UI only), no financial data leaks to unauthorized users  
**Scale/Scope**: Unified backend with two discrete frontends (`apps/storefront`, `apps/admin`)

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

- [x] **Server Authority Gate**: Is the Next.js frontend kept strictly as a presentation layer? Are all logic, pricing, inventory computations, and state mutations atomic and within Convex?
- [x] **Zero Floor & Concurrency Gate**: Can `real_stock` drop below zero in this design? Are stock deductions limited ONLY to the `CONFIRMED` state? Does the logic safely handle "Negative Concurrency Collisions"?
- [x] **RBAC & Financial Opacity Gate**: Are we using granular permission flags (e.g., `VIEW_FINANCIALS`) at the Convex mutation level rather than hardcoded roles? Are financial fields stripped from payloads for unprivileged users before leaving the server?
- [x] **Audit Gate**: Does this feature generate immutable audit logs for state, configuration, and permission changes?
