# Data Model: Catalog & Persistent Cart Engine

## Convex Schema Definitions

### Existing Tables (Modified)

#### `users` — Permission Extension

The `permissions` array union must be extended to include new catalog/inventory flags:

```typescript
permissions: v.array(
  v.union(
    // Existing
    v.literal("VIEW_FINANCIALS"),
    v.literal("VERIFY_PAYMENTS"),
    // New — Catalog & Inventory
    v.literal("MANAGE_CATEGORIES"),
    v.literal("MANAGE_PRODUCTS"),
    v.literal("MANAGE_DISPLAY_STOCK"),
    v.literal("ADJUST_REAL_STOCK"),
  )
),
```

#### `products` — Commercial Data Expansion

Existing `display_stock` and `real_stock` fields are unchanged. New commercial fields are added:

```typescript
products: defineTable({
  // --- Existing Fields (DO NOT MODIFY) ---
  // Virtual stock for storefront urgency display
  display_stock: v.number(),
  // Physical stock. Permitted to drop to -5 (Oversell Buffer). Must throw if below -5.
  real_stock: v.number(),

  // --- New Commercial Fields ---
  categoryId: v.id("categories"),           // Required — enforces parent-child integrity
  name_ar: v.string(),                      // Arabic product name (RTL storefront)
  name_en: v.string(),                      // English product name (LTR storefront)
  description_ar: v.optional(v.string()),   // Rich Arabic description
  description_en: v.optional(v.string()),   // Rich English description
  images: v.array(v.string()),              // Convex File Storage IDs
  selling_price: v.number(),               // Customer-facing price (EGP)
  cogs: v.optional(v.number()),            // Cost of Goods Sold — VIEW_FINANCIALS restricted
  status: v.union(                         // Publication FSM
    v.literal("DRAFT"),
    v.literal("PUBLISHED"),
  ),
  // Legacy field — may be removed in future if superseded by name_ar/name_en
  name: v.optional(v.string()),
  price: v.optional(v.number()),           // Legacy — superseded by selling_price
})
  .index("by_category", ["categoryId"])
  .index("by_status", ["status"]),
```

### New Tables

#### `categories`

```typescript
categories: defineTable({
  name_ar: v.string(),                       // Arabic category name
  name_en: v.string(),                       // English category name
  description_ar: v.optional(v.string()),
  description_en: v.optional(v.string()),
  thumbnailImageId: v.optional(v.string()),  // Convex File Storage ID for category thumbnail
  isActive: v.boolean(),                     // Soft-deletion flag — never hard-deleted
  slug: v.string(),                          // URL-friendly identifier (e.g., "smart-watches")
})
  .index("by_slug", ["slug"])
  .index("by_active", ["isActive"]),
```

**Integrity Rules:**
- `isActive = false` hides all child products from storefront browsing/search.
- Products with a parent category where `isActive = false` cannot be transitioned to `PUBLISHED`.
- Direct product URLs remain accessible (not 404) but render an "Unavailable" state.

#### `cart_sessions`

```typescript
cart_sessions: defineTable({
  sessionId: v.string(),       // UUID generated by frontend, stored in localStorage
  items: v.array(
    v.object({
      productId: v.id("products"),
      quantity: v.number(),
      addedAt: v.number(),     // Unix timestamp for abandoned cart analytics
    })
  ),
  lastUpdated: v.number(),     // Unix timestamp for TTL/cleanup logic
})
  .index("by_session", ["sessionId"]),
```

**Integrity Rules:**
- Cart additions do NOT affect `display_stock` or `real_stock` — no inventory reservation.
- Cart items are validated against live `display_stock` only during the pre-checkout `validateCart` call.
- Products deleted/unpublished since cart addition must be detected and reported at `validateCart` time.

---

#### `webhook_receipts`

```typescript
webhook_receipts: defineTable({
  rawPayload: v.any(),                          // Full raw payload from WhatsApp provider
  payloadHash: v.string(),                      // SHA-256 hash for idempotency check
  senderPhone: v.optional(v.string()),          // Extracted sender phone number
  extractedOrderCode: v.optional(v.string()),   // Short code parsed from message text (e.g., "ORD-123")
  mediaReferenceId: v.optional(v.string()),     // Convex File Storage ID for uploaded screenshot
  matchedOrderId: v.optional(v.id("orders")),  // Populated if order was found and matched
  processingStatus: v.union(
    v.literal("MATCHED"),
    v.literal("UNMATCHED"),       // No order found — queued for manual review
    v.literal("DUPLICATE"),       // Idempotency — identical payload already processed
    v.literal("INVALID"),         // Signature verification failed or malformed payload
  ),
  receivedAt: v.number(),         // Unix timestamp
})
  .index("by_hash", ["payloadHash"])
  .index("by_status", ["processingStatus"]),
```

## Convex Mutation & Query API Surface

### `categories.ts`

| Function | Type | Permission Required | Description |
|---|---|---|---|
| `createCategory` | mutation | `MANAGE_CATEGORIES` | Creates a new category with localized fields |
| `updateCategory` | mutation | `MANAGE_CATEGORIES` | Updates name, description, thumbnail of a category |
| `toggleCategoryActive` | mutation | `MANAGE_CATEGORIES` | Flips `isActive`; logs audit entry |
| `listActiveCategories` | query | none (public) | Returns all categories where `isActive = true` |
| `getCategoryById` | query | none (public) | Single category lookup by ID |

### `products.ts`

| Function | Type | Permission Required | Description |
|---|---|---|---|
| `createProduct` | mutation | `MANAGE_PRODUCTS` | Creates product; validates `categoryId` is active |
| `publishProduct` | mutation | `MANAGE_PRODUCTS` | Transitions DRAFT → PUBLISHED; validates parent category active |
| `unpublishProduct` | mutation | `MANAGE_PRODUCTS` | Transitions PUBLISHED → DRAFT |
| `getProduct` | query | none (public); `VIEW_FINANCIALS` for COGS | Returns product; strips `cogs` if no financial permission |
| `listProductsByCategory` | query | none (public) | Returns PUBLISHED products for active category |

### `cart.ts`

| Function | Type | Permission Required | Description |
|---|---|---|---|
| `addToCart` | mutation | none (guest) | Upserts item; validates `display_stock >= qty` |
| `removeFromCart` | mutation | none (guest) | Removes item from session cart |
| `updateCartQuantity` | mutation | none (guest) | Updates quantity; re-validates `display_stock` |
| `getCart` | query | none (guest) | Returns current cart items for a sessionId |
| `validateCart` | query | none (guest) | Strict per-item validation against live `display_stock` |
| `clearCart` | mutation | none (guest) | Empties cart (called by createOrder on success) |

### `webhooks.ts`

| Function | Type | Permission Required | Description |
|---|---|---|---|
| `processWebhookReceipt` | internalMutation | none (internal only) | Verifies idempotency, extracts order code, attaches media, logs |
| `attachReceiptManually` | mutation | `VERIFY_PAYMENTS` | Agent manually attaches receipt to a pending order; logs audit |

## State Transitions

### Product Publication FSM

```
DRAFT → PUBLISHED   (requires: MANAGE_PRODUCTS permission + parent category isActive = true)
PUBLISHED → DRAFT   (requires: MANAGE_PRODUCTS permission)
```

### Category Active State

```
isActive: true → false   (requires: MANAGE_CATEGORIES; cascades to hide all child products from storefront)
isActive: false → true   (requires: MANAGE_CATEGORIES; restores child product visibility)
```

### Webhook Receipt Processing

```
(incoming payload)
  → DUPLICATE    (payloadHash already exists in webhook_receipts)
  → INVALID      (signature verification failed or malformed)
  → UNMATCHED    (order code not found or order not in AWAITING_VERIFICATION)
  → MATCHED      (order found, media attached, audit logged)
```
